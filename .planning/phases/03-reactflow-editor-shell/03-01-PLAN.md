# 03-01-PLAN

## Objective
Create the first editable, persistent patch editor using React Flow.

## Deliverables
- React Flow canvas
- node palette
- custom nodes for the starter set
- node inspector
- typed edge validation in the UI
- patch save/load
- starter patch

## Atomic execution steps

### 1. Patch model
- Define the persisted graph document schema.
- Add patch metadata: id, name, version, created_at, updated_at.
- Add migration/version hooks before any real data ships.

### 2. Editor shell
- Add the React Flow canvas, controls, minimap, and basic layout.
- Register custom node components for the starter set.

### 3. Inspector and editing
- Build a property inspector for each starter node type.
- Synchronize node config changes back to the graph document.

### 4. Client-side validation
- Enforce simple port typing and connection rules.
- Flag missing required fields.

### 5. Persistence
- Save and load patch JSON.
- Provide a default starter patch that mirrors the hard-coded Phase 2 mapping path.

## Verification
- A user can create a graph visually.
- Invalid connections are blocked in the editor.
- Saving and reloading preserves the graph.
- The starter patch appears correctly with editable nodes.

## Exit criteria
- The graph can be authored visually.
- Patch documents are stable enough to feed the runtime phase.
- The old hard-coded mapping still exists for regression comparison.
