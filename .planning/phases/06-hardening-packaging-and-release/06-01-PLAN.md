# 06-01-PLAN

## Objective
Package, verify, and document the v1 release candidate.

## Deliverables
- sidecar-ready worker bundle
- integration test suite
- replay fixture set
- packaged build smoke checklist
- troubleshooting guide
- sample patches and demo instructions

## Atomic execution steps

### 1. Test harnesses
- Add unit tests for mapping functions and validators.
- Add replay-based runtime tests.
- Add integration tests for OSC packet generation and sending.

### 2. Worker packaging
- Package the worker as a deployable sidecar-compatible binary artifact.
- Verify start/stop behavior in packaged builds.

### 3. Failure handling
- Harden timeout handling, worker crash recovery, and status messaging.
- Ensure the UI never pretends tracking is healthy when it is not.

### 4. Release docs
- Write setup steps.
- Write "first five minutes" usage guide.
- Write troubleshooting for:
  - no camera preview
  - wrong camera selected
  - no OSC packets arriving
  - patch failed to load

### 5. Acceptance demo
- Run the complete user journey:
  - launch app
  - select camera
  - start tracking
  - load starter patch
  - observe mapped output
  - verify OSC at receiver

## Verification
- Packaged build launches.
- Worker starts in packaged build.
- Saved patch loads in packaged build.
- OSC packets reach a local receiver.
- Docs are enough that a new user can repeat the demo.

## Exit criteria
- v1 release candidate is real.
- Core workflow works outside dev mode.
- The next work belongs in backlog or milestone 2, not in endless pre-release tinkering.
