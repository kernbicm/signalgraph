# RESEARCH

## Why this phase exists
The difference between "cool prototype" and "usable tool" is usually calibration, monitoring, and persistence.

## Calibration approaches
- manual min/max entry
- learned min/max from a capture window
- optional reset to defaults
- optional invert and clamp behavior

## OSC usability requirements
- explicit host and port
- explicit address
- explicit payload type
- visible last-send timestamp
- packet monitor or log
- sane behavior when output is disabled

## Candidate preset templates
- Hand X to `/demo/hand/x`
- Wrist Y to `/demo/wrist/y`
- Gesture category to `/demo/gesture/name`
