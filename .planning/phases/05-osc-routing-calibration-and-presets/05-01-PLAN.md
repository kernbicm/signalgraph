# 05-01-PLAN

## Objective
Add the operational features that make patching and routing reliable in practice.

## Deliverables
- richer OSC output node
- packet monitor
- calibration workflow
- patch browser
- import/export
- starter templates
- multi-sink support

## Atomic execution steps

### 1. OSC node maturation
- Extend `osc_out` config with host, port, address, payload type, and enable state.
- Support multiple OSC output nodes per patch.

### 2. Packet monitor
- Add a diagnostics panel that shows outgoing packet history.
- Include failed send attempts and validation errors.

### 3. Calibration
- Add capture-based range learning.
- Apply captured ranges to map nodes or source-node calibration blocks.

### 4. Patch management
- Add create, duplicate, rename, delete, import, export.
- Persist patch metadata cleanly.

### 5. Starter templates
- Ship at least one patch template focused on the core user story:
  `MediaPipe value -> transform -> specific OSC address`

## Verification
- One patch can drive multiple OSC addresses.
- Calibration updates visibly affect mapped output.
- Import/export round-trips without corruption.
- Packet monitor reflects real outgoing traffic.

## Exit criteria
- A user can build and manage practical mappings without fighting the tool.
- The system tells the truth about outgoing OSC.
- The project is feature-complete enough for release hardening.
