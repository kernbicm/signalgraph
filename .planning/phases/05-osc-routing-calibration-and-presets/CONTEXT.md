# CONTEXT

## Goal
Make the system practical for real creative-coding use instead of only technically complete.

## Key user outcome
A user can pick a source value, calibrate it, map it, duplicate the patch, and route multiple outputs to explicit OSC addresses with confidence.

## Product rule
The app must tell the truth about what it is sending.
No silent failure, no fake "connected" states, no packet black box.

## Priority features
- richer OSC node config
- packet monitor
- calibration workflow
- preset management
- starter templates
