# 02-01-PLAN

## Objective
Build a hard-coded end-to-end slice from live tracking to a live OSC packet.

## Deliverables
- camera preview
- device selection
- one MediaPipe worker profile
- adapted signal namespace
- hard-coded mapping panel
- live OSC output
- health surfaces for worker and packet activity

## Atomic execution steps

### 1. Camera preview
- Add live preview to the Tauri frontend.
- Persist the user's last-selected device if possible.
- Surface permission errors clearly.

### 2. Worker inference loop
- Implement the initial Python tracking worker.
- Load one tracker profile.
- Emit compact telemetry frames on stdout or another narrow transport.

### 3. Signal adaptation
- Convert worker outputs into stable signal names.
- Publish a source explorer in the UI.
- Mark values as numeric, categorical, boolean, vector, or unsupported.

### 4. Hard-coded mapping path
- Add a simple panel that maps one selected numeric signal to one OSC output.
- Support manual min/max mapping and optional inversion.
- Send packets on every valid update.

### 5. Debug and recovery
- Show the last worker frame timestamp.
- Show last OSC packet sent.
- Surface worker restart controls.

## Files likely to change
- preview UI
- tracking controls
- worker process code
- telemetry parser
- signal adapter
- direct OSC mapping panel

## Verification
- Camera preview renders.
- Worker starts and stops reliably.
- At least one numeric signal updates live from a real tracker.
- The app sends a correct OSC packet to a local test target.
- The user can change the OSC address and observe the change in the packet monitor/receiver.

## Exit criteria
- The full product loop works once, end-to-end.
- One hard-coded mapping path exists and stays available as a regression check.
- The project is ready for a graph editor without any fake data illusions.
