# 04-01-PLAN

## Objective
Build the Rust runtime that evaluates saved graphs and drives output nodes from live source data.

## Deliverables
- graph validator
- runtime graph compiler
- topological sorter
- node evaluators for starter transform set
- runtime debug snapshots
- replay tests for deterministic evaluation

## Atomic execution steps

### 1. Validation layer
- Validate node existence and edge integrity.
- Enforce DAG-only graphs.
- Enforce port typing and required config presence.

### 2. Runtime compiler
- Convert patch JSON into an efficient runtime model.
- Precompute execution order.
- Prebind source node signal paths.

### 3. Node evaluators
Implement and test:
- `constant`
- `tracker_signal`
- `map_range`
- `clamp`
- `invert`
- `add`
- `multiply`
- `smooth`
- `deadzone`
- `threshold`
- `debug_meter`
- `osc_out` draft generation

### 4. Runtime integration
- Feed telemetry frames into the runtime.
- Emit live node outputs for debug inspection.
- Surface runtime errors and stale source states to the UI.

### 5. Replay testing
- Record a few telemetry fixtures.
- Run graphs against fixtures in tests and assert outputs.

## Verification
- A saved patch executes without the hard-coded path.
- Known input sequences produce expected mapped outputs.
- Runtime survives missing or stale source values.
- Runtime output values are inspectable in the UI.

## Exit criteria
- The visual graph actually drives behavior.
- Mapping math is tested and trusted.
- The project is ready for richer output and usability features.
